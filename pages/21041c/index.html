# 搭建私服

# docker-registry

  • 下载registry

    docker pull registry

  • 启动容器

    docker run -d -p 5000:5000 -v /tmp/myregistry/:/tmp/registry --privileged=true registry

  • 修改 /etc/docker/daemon.json

    {"insecure-registries":["0.0.0.0:5000"]}

  • 重启服务

    systemctl daemon-reload
systemctl restart docker

  • 执行命令

    docker tag myubuntu:1.1 0.0.0.0:5000/myubuntu:1.1
docker push 0.0.0.0:5000/myubuntu:1.1

  • 查看私服库的镜像

        curl -XGET http://0.0.0.0:5000/v2/_catalog

# Harbor

Harbor相比较于registry，提供了web管理界面，更详细的账户、权限管理，还可以配置漏洞扫描等工具

  1. 安装docker compose

    wget https://github.com/docker/compose/releases/download/v2.4.1/docker-compose-linux-x86_64
cp docker-compose-linux-x86_64 /usr/local/lib/docker/cli-plugins/docker-compose
chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

    测试安装

    docker compose version

  2. 下载Harbor

    wget https://github.com/goharbor/harbor/releases/download/v2.5.0/harbor-offline-installer-v2.5.0.tgz

  3. 解压harbor

    tar xf harbor-offline-installer-v2.5.0.tgz -C /usr/local/

  4. 配置harbor

    cp harbor.yml.tmpl harbor.yml
nano harbor.yml

    # Configuration file of Harbor

# The IP address or hostname to access admin UI and registry service.
# DO NOT use localhost or 127.0.0.1, because Harbor needs to be accessed by external clients.
hostname: 0.0.0.0 #改为域名或本机ip

# http related config
http:
  # port for http, default is 80. If https enabled, this port will redirect to https port
  port: 80

#将https相关证书注释掉
# https related config
#https:
  # https port for harbor, default is 443
  #port: 443
  # The path of cert and key files for nginx
  #certificate: /your/certificate/path
  #private_key: /your/private/key/path

# # Uncomment following will enable tls communication between all harbor components
# internal_tls:
#   # set enabled to true means internal tls is enabled
#   enabled: true
#   # put your cert and key files on dir
#   dir: /etc/harbor/tls/internal

  5. 运行安装脚本

    sudo ./install.sh --with-trivy --with-chartmuseum

  6. 让docker信任harbo

    修改 /etc/docker/daemon.json

    {"insecure-registries":["0.0.0.0"]}

  7. docker使用harbo仓库

    访问harbo的地址，默认账户：admin,密码：Harbo12345

    1. 新建项目

    1. 修改镜像标签符合仓库规范

      docker tag myubuntu:1.1 0.0.0.0/test/myubuntu:1.1

    2. docker login

      使用harbo的账户进行登录

    3. 上传镜像

      root@kiro:/etc/docker# docker push 0.0.0.0/test/myubuntu:1.1
The push refers to repository [0.0.0.0/test/myubuntu]

    4. 拉取镜像

      docker pull 0.0.0.0/test/myubuntu:1.1
