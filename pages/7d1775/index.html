# 面试题

# 一、Java 基础

1.JDK 和 JRE 有什么区别？ 
	JDK: JAVA开发者工具包
	JRE: JAVA运行环境
2.== 和 equals 的区别是什么？ 
	==为比较运算符，在基础数据类型中，==直接比较值，在引用类型中，== 比较的是对象的地址，equals是Object类中的方法，使用时依赖类的具体实现
	在比较int和Integer时，会出现一些其他状况,由于IntegerCache的存在，在比较 Integer a = 5 与 Integer b = 5时 a == b,a.equals(b))的结果都为true,是因为IntegerCache缓存了 -128 ~ 127 范围内的数，a 和 b指向 IntegerCache中的同一块地址
3.两个对象的 hashCode()相同，则 equals()也一定为 true，对吗？ 
	hashCode()相同时，equals()不一定相同，存在撞hash的可能
4.final 在 java 中有什么作用？ 
	final可以修饰 类、方法、变量；
	final修饰类时，类不可被继承；
	final修饰方法时，方法不可以被重写
	final修饰属性时，属性为常量，初始化后不可更改
5.java 中的 Math.round(-1.5) 等于多少？ 
	Math.round()中 首先给原来的数 +0.5,然后向下取整
	如-1.5，-1.5+0.5 = -1.0，向下取整后为-1.0
	-1.6，-1.6+0.5= -1.1， 向下取整后为-2.0
	1.5， 1.5+0.5 = 2.0，向下取整为2.0
	1.4， 1.4+0.5 = 1.9，向下取整为1.0
6.String 属于基础的数据类型吗？ 
	String是引用类型，不属于基本类型
	基础类型：byte,boolean,short,int,long,float,double,char
	对应的包装类：Byte,Boolean,Short,Integer,Long,Float,Double,Character
7.java 中操作字符串都有哪些类？它们之间有什么区别？ 
	有String,StringBuffer,StringBuilder三种
	String为不可变的，每次进行更改都需要重新分配内存，StringBuffer与StringBuilder则不需要
	StringBuffer为线程安全的，StringBuilder是线程不安全的
8.String str="i"与 String str=new String(“i”)一样吗？ 
	不一样，String str="i" i存在常量池中，str指向其地址，而String str=new String(“i”)则重新分配了内存
9.如何将字符串反转？ 
	1. 使用StringBuffer和StringBuilder的reverse()
	2. 通过charat()进行交换
	3. 通过collects类中的reversse()
10.String 类的常用方法都有那些？ 
	1. toString()
	2. charAt()
	3. indexOf()
	4. length()
	5. replace()
	6. replaceAll()
	7. lastIndexOf()
	8. split()
	9. substring()
	9. equals()
11.抽象类必须要有抽象方法吗？ 
	不需要，但是有抽象方法的一定是抽象类
12.普通类和抽象类有哪些区别？ 
	抽象类不可以被实例化
13.抽象类能使用 final 修饰吗？ 
	不能，抽象类需要被子类实现，不能用final修饰
14.接口和抽象类有什么区别？ 
	接口中的变量为public static 类变量，抽象类中的变量可以是普通成员变量
	接口可以被实现多个，抽象类只能被继承一个
15.java 中 IO 流分为几种？ 
	字节流：InputStream,OutputStream
	字符流：Writer,Reader
16.BIO、NIO、AIO 有什么区别？
17.Files的常用方法都有哪些？

# 二、容器

18.java 容器都有哪些？ 
    Collection: 
    	List: Vector, ArrayList,LinkedList
    	Set: HashSet,LinkedHashSet,TreeSet
    Map:
    	HashMap, LinkedHashMap, TreeMap, ConcurrentHashMap,HashTable,
19.Collection 和 Collections 有什么区别？ 
    Collection是一个接口，是Set,List 的父接口；Collections是工具类，提供了一系列工具用于集合，如排序，搜索
20.List、Set、Map 之间的区别是什么？ 
    List是有序集合，元素可重复；Set是无序集合，元素不可重复，LinkedHashSet按照插入顺序排序
    Map是键值对映射，Key 无序，不可重复
21.HashMap 和 Hashtable 有什么区别？ 
    HashMap是线程不安全的，允许key和value为 null
    HashTable是线程安全的，不允许为空
22.如何决定使用 HashMap 还是 TreeMap？ 
    TreeMap要求key实现compareable接口，默认按升序排序
    需要得到有序的结果时使用treemap
23.说一下 HashMap 的实现原理？ 
    HashMap使用数组加链表实现，进行put时，对key进行两次hash，然后与数组大小进行取模。如果两个key映射到同一个数组位置，则通过链表进行存储，在jdk1.8中，链表的长度大于8时，链表将转换为红黑树
24.说一下 HashSet 的实现原理？ 
    HashSet封装了一个HashMap
25.ArrayList 和 LinkedList 的区别是什么？ 
    ArrayList使用数组实现，随机访问效率高，插入删除效率低；LinkedList使用双向链表实现，随机访问效率低，增删效率高
26.如何实现数组和 List 之间的转换？ 
    Arrays.asList(),List.toArray
27.ArrayList 和 Vector 的区别是什么？ 
    Vector是线程安全的
28.Array 和 ArrayList 有何区别？ 
    Array不可扩容，ArrayList是动态数组
29.在 Queue 中 poll()和 remove()有什么区别？
30.哪些集合类是线程安全的？ 
    Vector,HashTable,ConcurrentHashMap

## 三、多线程
35.并行和并发有什么区别？
36.线程和进程的区别？ 
    进程是计算机分配资源的最小单位，线程是cpu执行的最小单位
37.守护线程是什么？ 
    thread在start之前，SetDeamon(true),守护线程在有其他线程存在时一直存活，在其他线程都结束后，守护线程将被关闭，如jvm的垃圾回收
38.创建线程有哪几种方式？ 
    继承thread类，实现callable,runnable接口，使用线程池
39.说一下 runnable 和 callable 有什么区别？ 
    runable没有返回值，callable有返回值，允许抛出异常
40.线程有哪些状态？ 
    	new,runnable,running,waiting,block
41.sleep() 和 wait() 有什么区别？ 
    sleep不会放开锁，属于Thread类的方法
    wait会防开锁，可进行线程通信，属于Object类的方法
42.notify()和 notifyAll()有什么区别？ 
    notify()随机唤醒一个wait的线程，可能导致死锁发生；
    notifyAll()唤醒所有线程
43.线程的 run()和 start()有什么区别？ 
    run()在当前线程执行方法，start()创建一个新线程执行方法
44.创建线程池有哪几种方式？ 
    Executors工具类，new threadpoolexcutor()
45.线程池都有哪些状态？ 
    running,shutdown,stop,terminated
46.线程池中 submit()和 execute()方法有什么区别？ 
    executor()返回类型为void,submit()返回future
47.在 java 程序中怎么保证多线程的运行安全？ 
    使用原子类，synchronized,volatile,Lock
48.多线程锁的升级原理是什么？ 
    无锁->偏向锁->轻量锁->重量级锁
49.什么是死锁？ 
    两个线程分别请求对方拥有的资源
50.怎么防止死锁？ 
    一段时间没有获取到锁后放弃自己的锁
51.ThreadLocal 是什么？有哪些使用场景？ 
    线程独有的空间，可以用来存放session
52.说一下 synchronized 底层实现原理？
53.synchronized 和 volatile 的区别是什么？ 
    synchronized是锁，volation保证可见性及避免代码重排
54.synchronized 和 Lock 有什么区别？

## 十四、RabbitMQ
135.rabbitmq 的使用场景有哪些？

136.rabbitmq 有哪些重要的角色？ 
    生产者，消费者，代理
137.rabbitmq 有哪些重要的组件？ 
    ConnectionFactory，channel,Exchange,Queue,RoutingKey,BindingKey
138.rabbitmq 中 vhost 的作用是什么？ 
    vhost,可以视为rabbitmq的虚拟机，里面有着独立的Exchange、Queue等，拥有者独立的权限控制和用户管理
139.rabbitmq 的消息是怎么发送的？

    1. 消息持久化
    2. Ack机制
    3. 集群镜像
    4. 消息补偿
141.rabbitmq 怎么避免消息丢失？
142.要保证消息持久化成功的条件有哪些？ 
    Exchange设置持久化
    Queue设置持久化
    Message 设置持久化
143.rabbitmq 持久化有什么缺点？ 
    降低吞吐量
144.rabbitmq 有几种广播类型？ 
    1. direct:默认消息发送方式，直接将消息发送给对方，如果有多个订阅者就轮询进行消息发送。消息中的路由键（routing key）如果和 Binding 中的 binding key 一致， 交换器就将消息发到对应的队列中，路由键与队列名完全匹配
    2. fanout:分发给与Exchange绑定的的队列上，不处理路由键
    3. topic: 通过对路由键进行模式匹配分发消息
145.rabbitmq 怎么实现延迟消息队列？ 
    TTL
    给quene和message设置x-message-tt，来控制消息生存时间，超时后，消息会变成dead letter
    DLX
    Queue可以配置x-dead-letter-exchange 和x-dead-letter-routing-key（可选）两个参数,如果队列内出现了dead letter，则按照这两个参数重新路由
    1. x-dead-letter-exchange：出现dead letter之后将dead letter重新发送到指定exchange
	2. x-dead-letter-routing-key：指定routing-key发送
146.rabbitmq 集群有什么用？ 
    高可用，高容量
147.rabbitmq 节点的类型有哪些？ 
    磁盘节点，内存节点
148.rabbitmq 集群搭建需要注意哪些问题？
149.rabbitmq 每个节点是其他节点的完整拷贝吗？为什么？ 
    不是
150.rabbitmq 集群中唯一一个磁盘节点崩溃了会发生什么情况？ 
    可以正常运行，但是不能改变任何东西
151.rabbitmq 对集群节点停止顺序有要求吗？ 
    先关闭内存节点，再关闭磁盘节点
