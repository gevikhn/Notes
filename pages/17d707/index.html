DockerFile

# DockerFile

# 格式

必须以FROM指令开始，一行以#符号开始，将忽略该行

# 保留字

  1. ENV：设置环境变量

    FROM busybox
ENV FOO=/bar
WORKDIR ${FOO}   # WORKDIR /bar
ADD . $FOO       # ADD . /bar
COPY \$FOO /quux # COPY $FOO /quux

  2. FROM：指定一个基础镜像作为模板，必须在文件中第一条指令

  3. MAINTAINER：作者

  4. RUN：在build时会执行的命令，可用shell格式

  5. EXPOSE：指定容器暴漏的端口

  6. WORKDIR：工作目录，进入容器时的默认路径

  7. USER：指定该镜像以哪种身份的用户运行，默认为ROOT

  8. ADD：向镜像中添加文件，支持url

    存在以下两种方式

    ADD [--chown=<user>:<group>] <src>... <dest>
ADD [--chown=<user>:<group>] ["<src>",... "<dest>"]

  9. COPY：将文件拷贝到镜像中

    COPY [--chown=<user>:<group>] <src>... <dest>
COPY [--chown=<user>:<group>] ["<src>",... "<dest>"]

  10. VOLUME：添加容器卷

  11. CMD：容器启动后执行命令，会被docker run 后的命令覆盖

  12. ENTRYPOINT：和CMD命令作用相似，不会被docker run后的命令覆盖

    ENTRYPOINT和CMD一起使用时，CMD的内容将作为参数传递给ENTRYPOINT

例：生成一个包含jdk的镜像

FROM ubuntu
MAINTAINER nanami<ciallomino@gmail.com>

ENV MYPATH /usr/local
WORKDIR $MYPATH

RUN mkdir /usr/local/java

RUN apt update
RUN apt install -y nano
RUN apt install -y net-tools

ADD openjdk-17.0.2_linux-x64_bin.tar.gz /usr/local/java

ENV JAVA_HOME /usr/local/java/jdk-17.0.2
ENV CLASSPATH $JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$CLASSPATH
ENV PATH $JAVA_HOME/bin:$PATH

EXPOSE 80

build:

docker build -t myos:1.0 .
