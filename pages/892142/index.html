# Controller

  • 在集群上管理和运行容器的对象
  • pod是通过Controller来实现各种操作的，如弹性伸缩、滚动升级
  • pod和Controller通过label来建立联系

# deployment

  • 部署无状态应用
  • 管理pod和Replicaset副本
  • 部署，滚动升级等功能
  • 常用于部署web服务/微服务

# 创建deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

  • 该配置文件将创建一个叫nginx-deployment的deployment，并由deployment创建3个Pod副本（由 replicas 字段标明）

  • selector 字段定义 Deployment 如何查找要管理的 Pods

  • template 字段包含以下子字段：

    • Pod 被使用 .metadata.labels 字段打上 app: nginx 标签。
    • Pod 模板规约（即 .template.spec 字段）指示 Pods 运行一个 nginx 容器， 该容器运行版本为 1.14.2 的 nginx Docker Hub (opens new window)镜像。
    • 创建一个容器并使用 .spec.template.spec.containers[0].name 字段将其命名为 nginx
  1. 通过以下命令创建deployment
kubectl apply -f nginx-deployment.yml
  1. 运行 kubectl get deployments 检查 Deployment 是否已创建
kubectl get deployments
NAME               READY   UP-TO-DATE   AVAILABLE   AGE
nginx-deployment   0/3     0            0           1s
  1. 要查看 Deployment 上线状态，运行 kubectl rollout status deployment/nginx-deployment
kubectl rollout status deployment/nginx-deployment
Waiting for rollout to finish: 2 out of 3 new replicas have been updated...
deployment "nginx-deployment" successfully rolled out
  1. 几秒钟后再次运行 kubectl get deployments
kubectl get deployments
NAME               READY   UP-TO-DATE   AVAILABLE   AGE
nginx-deployment   3/3     3            3           18s
  1. 要查看 Deployment 创建的 ReplicaSet（rs），运行 kubectl get rs
NAME                          DESIRED   CURRENT   READY   AGE
nginx-deployment-75675f5897   3         3         3       18s
  1. 要查看每个 Pod 自动生成的标签，运行 kubectl get pods --show-labels。
kubectl get pods --show-labels
NAME                                READY     STATUS    RESTARTS   AGE       LABELS
nginx-deployment-75675f5897-7ci7o   1/1       Running   0          18s       app=nginx,pod-template-hash=3123191453

# 更新deployment

仅当 Deployment Pod 模板（即 .spec.template）发生改变时，例如模板的标签或容器镜像被更新， 才会触发 Deployment 上线。其他更新（如对 Deployment 执行扩缩容的操作）不会触发上线动作。

  1. 先更新 nginx Pod 以使用 nginx:1.16.1 镜像，而不是 nginx:1.14.2 镜像。

    kubectl set image deployment.v1.apps/nginx-deployment nginx=nginx:1.16.1

    等效于

    kubectl set image deployment/nginx-deployment nginx=nginx:1.16.1

    输出：

    deployment/nginx-deployment image updated

  2. 查看上线状态，运行

    kubectl rollout status deployment/nginx-deployment

    输出：

    Waiting for rollout to finish: 2 out of 3 new replicas have been updated...

    或

    deployment "nginx-deployment" successfully rolled out

# 回滚Deployment

当回滚到较早的修订版本时，只有 Deployment 的 Pod 模板部分会被回滚。

因为出错导致上线进程停止，如设置nginx版本时，将nginx:1.16.1设置为nginx:1.161：

kubectl set image deployment/nginx-deployment nginx=nginx:1.161 --record=true

检查上线状态：

kubectl rollout status deployment/nginx-deployment

输出：

Waiting for rollout to finish: 1 out of 3 new replicas have been updated...

检查 Deployment 上线历史

  1. 检查 Deployment 修订历史
kubectl rollout history deployment/nginx-deployment

输出：

deployments "nginx-deployment"
REVISION    CHANGE-CAUSE
1           kubectl apply --filename=https://k8s.io/examples/controllers/nginx-deployment.yaml
2           kubectl set image deployment/nginx-deployment nginx=nginx:1.16.1
3           kubectl set image deployment/nginx-deployment nginx=nginx:1.161

  1. 查看修订历史的详细信息

    kubectl rollout history deployment/nginx-deployment --revision=2

**回滚到之前的修订版本 **

  1. 撤消当前上线并回滚到以前的修订版本

    kubectl rollout undo deployment/nginx-deployment

    回滚到指定版本

    kubectl rollout undo deployment/nginx-deployment --to-revision=2

    回滚命令详细信息：kubectl rollout (opens new window)

  2. 检查回滚是否成功以及 Deployment 是否正在运行：

    kubectl get deployment nginx-deployment

  3. 获取 Deployment 描述信息

    kubectl describe deployment nginx-deployment

    输出：

# 缩放Deployment

执行

kubectl scale deployment/nginx-deployment --replicas=10

如果启用了Pod的水平自动缩放，可以为Deplyment设置自动缩放器，并基于现有 Pod 的 CPU 利用率选择要运行的 Pod 个数下限和上限。

kubectl autoscale deployment/nginx-deployment --min=10 --max=15 --cpu-percent=80
